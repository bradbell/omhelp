Description
This section contains a list of changes to OMhelp during 2019 (in reverse order by date). Its purpose is to assist you in learning about changes between versions.

19-12-07
Add another improvement to the _search utility. To be specific, hitting return or double clicking in the section tag area will also go to the selected section.

19-10-12
  1. Add the cref wish list item.
  2. Deprecate not using the -noframe command line argument. Date this deprecation the same day that frames were deprecated.


19-09-30
Improve the _search utility. To be specific, it shows the title and section tag in separate areas and hides the other keywords. In addition, hitting return or double clicking in the title area, in addition to choosing the Goto button, will go to the selected section.

19-09-29
The cend and rend commands have been deprecated .

19-09-28
  1. The path command has been deprecated .
  2. Put the whats new for 2005 through 2018 in separate files; see previous years above and remove the corresponding wish_list item.


19-08-29
It is not longer necessary to specify the boost_prefix for the CMake command because if is determined automatically using the CMake find_package command. This change fixed the install on Ubuntu which was broken because Ubuntu-18 puts the boost libraries in a sub directory corresponding to the architecture; e.g, 
 
    /usr/lib/x86_64-linux-gnu/libboost_regex.so

19-08-28
There is a problem installing on Ubuntu that needs to be fixed.

19-06-27
  1. There was a problem with the drop down navigation menus when a section tag had a character that could not be used as a Java script identifier; e.g., the period. This has been fixed.
  2. An extra restriction was added for the value of a section tag . To be specific, it cannot contain the sub-string __ (two underbars).


19-02-02
Add some items to the wish_list .

19-02-01
Fix bug whereby some shared navigation drop down menus were missing (see change on 19-01-29).

19-01-30
  1. A default default label was added for each navigation type.
  2. The OMhelp documentation now uses jump tables and the example navigation links were changed.
  3. The jump tables were missing at the top of the _index and _reference . This has been fixed.


19-01-29
  1. The drop down navigate links of type Down_up_i , with label label _up_i were identical for every child section. This has been replace by a single Java script file for the identical cases. This reduces the size of the files and the number of difference in gh-pages branches used to display omhelp documentation. (The corresponding 19-01-26 wish list item was removed).
  2. White space is now forbidden in the cross reference tag specified by the begin command. In addition, the restriction that _links and _frame cannot be part of the tag was added to the documentation.
  3. A special case where the navigation label is empty was added. This was later changed to a default label for each navigation type. This encourages a more efficient representation.


19-01-26
Add a navigation wish list item.

19-01-18
  1. There were two bugs in the optional jump_table at the top of a section. The first bug caused an xml format error when using jump tables with xml . The other bug did not indent sub headers below headers that started with the character '0'. These bugs have been fixed.
  2. There was an error message bug related to the verbatim command, when a new line was placed directly after the command. To be more specific, the new line was counted twice and the line number in subsequent error messages was larger than it should have been. This was also true for other commands that read from a different file; e.g., srcfile . This bug has been fixed.
